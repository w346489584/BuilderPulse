The loud AI story is Karpathy joining Anthropic and Google changing the search box. The builder signal is uglier and more urgent: [Mini Shai-Hulud Strikes Again](https://safedep.io/mini-shai-hulud-strikes-again-314-npm-packages-compromised/) says one npm account pushed 637 malicious versions across 317 packages in 22 minutes, and the payload can restart itself through Claude Code and Codex session hooks. A session hook is a script that runs automatically when a coding assistant starts; today that script became a supply-chain target.

**What are teams doing today?** They run `npm install`, open their editor, start a coding assistant, and assume the repo is clean because tests still pass.

**How big is the sample?** The compromise touched packages with millions of monthly downloads, drew 277 Hacker News comments, and named `size-sensor`, `echarts-for-react`, and `@antv` packages as affected.

**Why can an indie win?** Big security suites watch endpoints broadly; a solo builder can ship the narrow report that checks AI-session hooks, editor auto-runs, and changed workflows first.

The dirty work is not another malware scanner. It is reading package changes, `.github/workflows`, editor tasks, assistant startup files, and exposed token patterns, then handing the maintainer one page: what auto-runs, what changed, and what to revoke first.
