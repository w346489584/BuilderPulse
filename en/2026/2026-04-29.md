**Takeaway**: Use cooling terms for comparison context, not headline positioning; current replacement nouns give better distribution.

**Counter-view**: Cooling search momentum can still hide large installed bases, and installed bases often pay better than trend chasers.

### With 2 hours today or a full weekend, what should I build?

**🔍 Signal**: [Ghostty is leaving GitHub](https://mitchellh.com/writing/ghostty-leaving-github) drew 532 comments, GitHub availability drew 208 comments, and GitHub migration links appeared across Hacker News and Lobsters.

**In plain English:** Maintainers are asking whether they can leave GitHub without losing the work around the code.

**Best 2-hour build**: **ForgeExit Map** — a GitHub migration-readiness report for maintainers. The MVP asks for a public repo URL, then prints a Markdown report covering GitHub Actions workflows, issue templates, labels, pull request references, release artifacts, branch protections, badges, documentation links, project-board usage, and which parts can move cleanly to Codeberg, Forgejo, or a self-hosted forge.

**Why this wins today**: the buyer is visible and software-native. Ghostty's maintainer is not a random user; he is an experienced open-source operator with 18 years of GitHub habit. The comments show the real problem: @infogulch says Git is distributed but issues and pull requests are painful to lose; @nextaccountic points to git-bug as one possible way to store context; @nomilk asks whether GitHub Actions is the main pain. **Why not the other two**: an Android side-loading policy checker has a huge 468-comment discussion, but banking apps, device vendors, and regulation make validation slower. An AI code-ownership scanner is important, but legal conclusions require expertise and will be harder to sell in a two-hour MVP.

**Weekend expansion**: add private-repo login, Codeberg and Forgejo destination checks, a GitHub Actions replacement table, issue export, badge rewrite suggestions, and a $19 one-time downloadable migration packet for maintainers.

**Fastest validation step**: If you want to validate this today, start with three public repos that already discuss leaving GitHub, run a manual audit, publish the before/after migration checklist, and ask maintainers which line item scares them most.

**Takeaway**: Ship ForgeExit Map today; the job is concrete, the buyer is public, and the first version is a report, not a platform.

**Counter-view**: Most projects will stay because their contributors, stars, and automations already live on GitHub.

### What pricing and monetization models are worth studying?

**Takeaway**: Start with one paid report when the job is occasional; move to subscription only when the buyer needs recurring monitoring.

**Counter-view**: One-time reports can create lumpy revenue, so the follow-up workflow must be designed before acquisition costs rise.
