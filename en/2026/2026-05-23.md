**🔍 Signal**: The practical downgrades today are [yt-dlp limiting and deprecating Bun support](https://github.com/yt-dlp/yt-dlp/issues/16766), Google’s Antigravity IDE changing into a prompt-first experience, Google expanding Search ad formats, and Railway’s unresolved trust discussion after a Google Cloud suspension.

**In plain English:** A product does not need to disappear to break a workflow; changing the interface or support promise is enough.

The clearest downgrade is not a shutdown; it is support friction. When a project says a runtime is no longer worth first-class support, the market hears “fast is not free.” That hurts any team that adopted a runtime for speed without budgeting maintenance.

The Antigravity story is the workflow version of the same problem. Railway’s Google Cloud thread is not new enough to headline today, but it remains a cautionary example of platform-dependence risk.

**Takeaway**: Track support promises as product surface; if a runtime, IDE, or platform changes defaults, users need a migration note before the outage.

**Counter-view**: Some downgrades are rational maintenance choices, not betrayals, and the short-term anger may fade after migration paths settle.
